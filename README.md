# zhweb-ng28-journal

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，主要用于存放与“zhweb-ng28”主题相关的静态 HTML 页面资源，方便历史页面的查阅与维护。

## 项目简介

zhweb-ng28-journal 旨在收集、整理并发布一系列独立的 HTML 页面。这些页面可能涵盖不同的内容主题，但均以“zhweb-ng28”作为统一标识。仓库本身不提供动态服务，仅作为静态页面的存档与展示用途。

## 目录说明

仓库根目录下包含以下内容：

- `index.html` — 仓库主页面，可展示页面索引或导航信息。
- `pages/` — 存放所有独立 HTML 页面的目录，每个页面以独立的 HTML 文件形式存在。
- `assets/` — 存放页面所需的静态资源（如 CSS、JavaScript、图片等）。
- `README.md` — 本文件，提供仓库的基本说明与使用指引。

## 页面归档说明

所有 HTML 页面均以静态文件形式存储，不依赖后端服务。页面内容可能包含文本、链接、样式及交互元素，但均不涉及任何商业推广或具体域名指向。

归档原则：
- 每个页面独立成文件，文件名清晰反映页面主题。
- 页面之间无强耦合，可以单独访问。
- 资源文件统一放置在 `assets/` 目录下，方便管理。

## 维护说明

本仓库由维护者不定期更新。新增页面或更新现有页面时，请遵循以下规范：

1. 新页面文件放入 `pages/` 目录，并确保文件名唯一。
2. 若页面需要使用自定义资源，请将资源放入 `assets/` 下的对应子目录。
3. 修改 `index.html` 可更新页面索引。
4. 提交前请检查页面是否能在本地浏览器中正常显示。

欢迎通过 Issue 提交反馈或建议，但请注意本仓库不涉及任何商业或营销行为。
